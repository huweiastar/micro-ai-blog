**Tech Stack:** npm workspaces · Next 16 · 现有 systemd/nginx/deploy.sh

**前置约定：** 隔离分支 `feat/monorepo-a1`；动手前 `git tag pre-monorepo-a1`（A4/A5 之外的回滚锚点）。每任务末尾 commit。验证三件套：`npm run type-check` `npm run lint` `npm run build`（隔离 distDir，勿污染生产 `.next`，见 incident 记忆）。

### Task 3: 引入路径助手（仍指向当前根，零行为变化）

**Files:**
- Create: `lib/paths.ts`

- [ ] **Step 1: 失败测试 `lib/paths.test.ts`**
```ts
import { describe, it, expect } from "vitest";
import { contentDir, dataDir } from "./paths";
import path from "path";

describe("paths", () => {
  it("contentDir 默认 = repoRoot/content", () => {
    expect(contentDir()).toBe(path.join(process.cwd(), "content"));
  });
  it("dataDir 默认 = repoRoot/data", () => {
    expect(dataDir()).toBe(path.join(process.cwd(), "data"));
  });
  it("尊重 CONTENT_DIR 覆盖", () => {
    process.env.CONTENT_DIR = "/tmp/x/content";
    expect(contentDir()).toBe("/tmp/x/content");
    delete process.env.CONTENT_DIR;
  });
});
```
- [ ] **Step 2: 跑测确认失败** `npx vitest run lib/paths.test.ts`
- [ ] **Step 3: 实现 `lib/paths.ts`**
```ts
import path from "path";

/** 内容目录：默认 <仓库根>/content，可用 CONTENT_DIR 覆盖（monorepo 下指向共享根）。 */
export function contentDir(): string {
  return process.env.CONTENT_DIR || path.join(process.cwd(), "content");
}

/** 数据目录：默认 <仓库根>/data，可用 DATA_DIR 覆盖。 */
export function dataDir(): string {
  return process.env.DATA_DIR || path.join(process.cwd(), "data");
}
```
- [ ] **Step 4: 跑测确认通过** `npx vitest run lib/paths.test.ts`
- [ ] **Step 5: Commit** `git add lib/paths.ts lib/paths.test.ts && git commit -m "feat(paths): 内容/数据目录解析助手（默认仓库根，支持 env 覆盖）"`
