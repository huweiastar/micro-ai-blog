# 架构拆分 A2：脚手架 manager + 证明「写→revalidate」链路 Implementation Plan

> 逐任务实施，每任务末尾 commit。**A2 全程本地，不部署、不碰生产**（manager 部署属 A5）。

**Goal:** 在 monorepo 内新建 `apps/manager`（独立 Next app），用共享的 `@pkg/auth` 做登录鉴权，并打通「manager 触发 → blog 失效缓存」的 revalidate 控制链路。

**Architecture:** 抽出第一个共享包 `packages/auth`（自洽的 session-token）；blog 改为从包导入。manager 以 cookie + ADMIN_PASSWORD 鉴权（复用 @pkg/auth 的 sign/verify）。blog 新增 token 保护的 `/api/revalidate`。manager 一个受保护页里触发它，验证跨 app 控制链路。

**范围内**：包抽取(仅 auth/session-token)、manager 骨架、登录、blog /api/revalidate、manager 触发 demo、本地双 app 验证。
**范围外（留 A3/A4/A5）**：抽 content/db 包、迁移真正的 /admin 与写接口、真实内容写入、manager 部署/nginx 子域。

**前置**：分支 `feat/monorepo-a2`。

---

### Task 1: 抽出 packages/auth（session-token 自洽部分）
- Create: `packages/auth/package.json`（name `@pkg/auth`，main 指向 src）
- Create: `packages/auth/src/session-token.ts`（= 现 apps/blog/lib/session-token.ts 内容）
- Create: `packages/auth/tsconfig.json`
- Modify: `apps/blog/lib/auth.ts`、`apps/blog/middleware.ts`、其它从 `./session-token` 导入处 → 改为 `@pkg/auth`
- Delete: `apps/blog/lib/session-token.ts`
- 步骤：建包 → 把 blog 的 session-token 导入改为 `@pkg/auth` → `npm install`(链接包) → `npm run type-check -w @app/blog` + `npm run test` + 隔离构建，确认 blog 行为不变 → commit。

### Task 2: 脚手架 apps/manager 骨架
- Create: `apps/manager/package.json`（name `@app/manager`，deps: next/react/react-dom/@pkg/auth）
- Create: `apps/manager/next.config.mjs`、`tsconfig.json`、`postcss.config.mjs`、`tailwind.config.ts`、`app/globals.css`、`app/layout.tsx`、`app/page.tsx`（"管理端首页 · 占位"）
- 步骤：建文件 → `npm install` → `NEXT_DIST_DIR=.next.verify npm run build -w @app/manager` 成功 → commit。

### Task 3: manager 登录鉴权（@pkg/auth）
- Create: `apps/manager/middleware.ts`（校验 `admin_session` cookie via @pkg/auth verifyToken，未登录跳 /login）
- Create: `apps/manager/app/login/page.tsx` + `apps/manager/app/api/auth/login/route.ts`（校验 ADMIN_PASSWORD → signToken → set cookie）
- 步骤：构建通过 → 本地起 manager，未登录访问 / 跳 /login，登录后可进 → commit。

### Task 4: blog 新增 /api/revalidate（token 保护）
- Create: `apps/blog/app/api/revalidate/route.ts`（POST，校验 `x-revalidate-token` == `REVALIDATE_TOKEN` env，按 body.paths 调 `revalidatePath`）
- 步骤：type-check + 隔离构建 → commit。

### Task 5: manager 触发 demo + 本地双 app 验证
- Create: `apps/manager/app/api/revalidate-ping/route.ts`（受登录保护，POST blog `${BLOG_ORIGIN}/api/revalidate` 带 token + paths=["/"]）
- Modify: `apps/manager/app/page.tsx` 加按钮触发
- 步骤：本地同时起 blog(:3000 或 3001 隔离) 与 manager(:3002)，登录后点按钮 → 返回 200、blog 端 revalidate 生效 → commit。
- 终验：两 app 各自 type-check/lint/build；blog 全测仍过。

## 范围与回滚

- REVALIDATE_TOKEN / BLOG_ORIGIN 走 .env.local（本地）。
